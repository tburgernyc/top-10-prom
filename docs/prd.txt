### Top 10 Prom Phase 1 Digital Showroom (Master Specification v3.3.0)

**1. Executive Summary & Strategic Objectives**
The platform is an institutional-grade Digital Showroom designed to pivot from traditional e-commerce to an intent-capture and lead-generation model. The architecture relies on a Serverless, Edge-First infrastructure using Next.js 16 and the Vercel Edge Network, entirely circumventing Docker. A "Toggleable Commerce Pattern" allows future Stripe-backed e-commerce via a `COMMERCE_ENABLED` env variable.

**2. Core Technology Stack**
* Runtime: Node.js 24.x LTS.
* Framework: Next.js 16 (App Router, Turbopack). The `src/` directory is strictly prohibited.
* UI & Styling: React 19, Tailwind CSS 3.4.17, Motion 12 (imports strictly from `motion/react`).
* Database: Supabase CLI-managed PostgreSQL.
* AI Integrations: Google Vertex AI for Virtual Try-On, and Google Gemini 2.5 Flash for the "Aria" Concierge.

**3. Database Architecture & Security (Supabase)**
* Core Tables: profiles, dresses, wishlist, availability_inquiries.
* Shadow Cart: `fitting_room_sessions` tracking unauthenticated JSONB intent.
* Security (RLS): Strict RLS enforced via `(select auth.uid())`.

**4. Design System ("Dark Luxury") & Motion**
* Palette: Onyx (#050505), Gold (#D4AF37), Platinum (#C0C0C0), Ivory (#F5F5F5).
* UI: 5 glassmorphism variants, bespoke 8px dot/32px ring cursor (0.12s lerp).
* Motion: Lenis smooth scroll engine and Motion 12 AnimatePresence for 350ms page wipes. All images AVIF/WebP.

**5. AI Features & Server Actions**
* Aria AI Concierge: `app/api/chat/route.ts` via Vercel AI SDK and Gemini 2.5 Flash. Strictly non-transactional (no "Cart"/"Buy").
* Virtual Try-On: Proxy at `app/api/tryon/route.ts` with Vertex AI.
* Booking: 5-step interactive form built with React 19 Server Actions, synced with Google Calendar and Resend.

**6. Guardrails**
* TaskMaster must ensure `.claude/hooks.json` intercepts all commits.
